## What it does

- **Library** вҖ” `compress(messages)` in Python or TypeScript, inline in any app
- **Proxy** вҖ” `headroom proxy --port 8787`, zero code changes, any language
- **Agent wrap** вҖ” `headroom wrap claude|codex|cursor|aider|copilot` in one command
- **MCP server** вҖ” `headroom_compress`, `headroom_retrieve`, `headroom_stats` for any MCP client
- **Cross-agent memory** вҖ” shared store across Claude, Codex, Gemini, auto-dedup
- **`headroom learn`** вҖ” mines failed sessions, writes corrections to `CLAUDE.md` / `AGENTS.md`
- **Reversible (CCR)** вҖ” originals never deleted; LLM retrieves on demand

### Codex + Cognitive Context Manager (CCM)

Headroom is compatible with CCM and can run beside it in the same Codex install. A good default split is:

- CCM stays the durable project-memory layer: working context, decisions, open loops, and stale-memory hygiene.
- Headroom handles the local proxy, request compression, CCR `headroom_compress`/`headroom_retrieve`, and session stats.
- Both expose MCP tools; use CCM when you need project state and use Headroom when a compressed marker needs expansion or when you want proxy/compression metrics.

This repo includes a Codex plugin skeleton for that pairing: `.codex-plugin/plugin.json`, `.mcp.json`, `hooks/hooks.json`, and `skills/headroom/SKILL.md`. Install `headroom-ai[all]` so the `headroom` command is on `PATH`, enable the plugin in Codex, and keep CCM installed as the default project memory provider.

## When to use В· When to skip

**Great fit if youвҖҰ**
- run AI coding agents daily and want savings without changing your code
- work across multiple agents and want shared memory
- need reversible compression вҖ” originals always retrievable via CCR

**Skip it if youвҖҰ**
- only use a single provider's native compaction and don't need cross-agent memory
- work in a sandboxed environment where local processes can't run

<details>
<summary><b>What's inside</b></summary>

- **SmartCrusher** вҖ” universal JSON: arrays of dicts, nested objects, mixed types.
- **CodeCompressor** вҖ” AST-aware for Python, JS, Go, Rust, Java, C++.
- **Kompress-base** вҖ” our HuggingFace model, trained on agentic traces.
- **Image compression** вҖ” 40вҖ“90% reduction via trained ML router.
- **CacheAligner** вҖ” stabilizes prefixes so Anthropic/OpenAI KV caches actually hit.
- **IntelligentContext** вҖ” score-based context fitting with learned importance.
- **CCR** вҖ” reversible compression; LLM retrieves originals on demand.
- **Cross-agent memory** вҖ” shared store, agent provenance, auto-dedup.
- **SharedContext** вҖ” compressed context passing across multi-agent workflows.
- **`headroom learn`** вҖ” plugin-based failure mining for Claude, Codex, Gemini.

</details>

<details>
<summary><b>Pipeline internals</b></summary>

Headroom exposes one stable request lifecycle across `compress()`, the SDK, and the proxy:

`Setup` вҶ’ `Pre-Start` вҶ’ `Post-Start` вҶ’ `Input Received` вҶ’ `Input Cached` вҶ’ `Input Routed` вҶ’ `Input Compressed` вҶ’ `Input Remembered` вҶ’ `Pre-Send` вҶ’ `Post-Send` вҶ’ `Response Received`

- **Transforms** do the work: CacheAligner, ContentRouter, SmartCrusher, CodeCompressor, Kompress-base, IntelligentContext / RollingWindow.
- **Pipeline extensions** observe or customize lifecycle stages via `on_pipeline_event(...)`.
- **Compression hooks** sit alongside the canonical lifecycle as an additional extension seam.
- **Proxy extensions** remain the server/app integration seam for ASGI middleware, routes, and startup policy.

Provider and tool-specific behavior lives under `headroom/providers/` so core orchestration stays focused on lifecycle, sequencing, and policy.

- **CLI/tool slices**: `headroom/providers/claude`, `copilot`, `codex`, `openclaw`
- **Provider runtime slices**: `headroom/providers/claude`, `gemini`, plus shared backend/runtime dispatch in `headroom/providers/registry.py`
- **Core files stay orchestration-first**: `wrap.py`, `client.py`, `cli/proxy.py`, and `proxy/server.py` delegate provider-specific env shaping, API target normalization, backend selection, and transport dispatch.

</details>
